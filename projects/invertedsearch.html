Core Features

Database Creation

Creates inverted index from multiple text files with word frequency tracking across files.

Fast Word Search

O(1) average case search using hash tables to find words and their file occurrences instantly.

Dynamic Updates

Add new files to existing database without rebuilding the entire index from scratch.

Database Persistence

Save and load database to/from files for reuse across program executions.

Visual Display

Display entire database with words, file counts, and occurrence frequencies in each file.

Memory Management

Proper cleanup of all dynamically allocated memory to prevent leaks.

Key Challenges & Solutions

Fast Search

Challenge: Searching through all files for each word is O(n×m). Solution: Implemented hash table for O(1) average case lookup.

Complex Data Structure

Challenge: Managing nested linked lists (hash table → main nodes → sub nodes). Solution: Careful pointer management and modular functions.

Word Normalization

Challenge: "Hello" and "hello" should be treated as same word. Solution: Convert all words to lowercase before processing.

Compilation & Usage

# Compile all source files
gcc main.c invert.c assist.c -o inverted

# Run with text files as arguments
./inverted file1.txt file2.txt file3.txt

3. Display Database - Show entire index

6. Exit - Cleanup and exit
